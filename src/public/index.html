How it works

The Atomic Bid — engineered for correctness

1

Bid comes in

User places a bid. Server validates auction is active and bid is higher than current price.

2

Row-level lock

PostgreSQL locks the auction row with SELECT FOR UPDATE. No other request can touch it.

3

Atomic commit

Bid inserted and price updated in one transaction. Either both happen or neither does.

4

Live broadcast

Socket.io broadcasts new price to all connected users in under 100ms. No refresh needed.

5

Cache invalidated

Redis cache cleared instantly so next request fetches fresh data from PostgreSQL.

6

Winner declared

When timer hits zero, the highest bidder wins. One winner, always. Guaranteed.
